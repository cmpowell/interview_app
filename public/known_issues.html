Known Issues

This page describes known issues in InterviewApp.

Issues related to functionality

  1. No method is available to delete recipes
  2. Very long recipes will generate an error page (too long to be stored in db)
  3. At login redirect goes to the same page, not always the page you originally requested
  4. Emails are not required to look like emails (e.g., "imanemail" is allowed)
  5. The check that emails are unique is case sensitive (e.g., foo@BAR.com does not match FOO@bar.com)
  6. Titles must be between 4 and 8 characters long, which seems overly restrictive
  7. If you enter a non-integer for servings, it fails silently and shows 0 thereafter
  8. Clicking on a recipe takes you to the page but not to that recipe on the page
Non-Functional Issues
  1. If two users submit simultaneously, a race in email validation vs db update might allow non-unique emails
  2. Any user can edit any other user's profile
  3. An active session is not ended when the user closes the browser
  4. The help is more insulting than helpful
Look and feel problems
  1. No way to delete recipes
  2. Very long words aren't handled well in the UI
  3. Updating a profile and failing produces a different browser title than original update
  4. On IE, the signup button doesn't have rounded corners
